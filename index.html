Gestion de contacts

Notre application (en C#) de gestion de contacts est une application basique liée aux données. Vous pouvez l’utiliser pour créer de nouveaux contacts, éditer des contacts existants et en supprimer.
Nous allons donc suivre ces étapes :
1. Couche Model : Création d’une application de type Bibliothèque de classes contenant les entités de base de l’application (Contact, …)
2. Couche View : Création d’une application WPF (.exe) avec une référence vers la bibliothèque de classes précédente.
3. Générer une base de données dans SQL Serveur.
4. Créer une structure d’application (CReate Update Delete) permettant de réaliser les opérations suivantes :
A. Ajout d’un nouveau contact
B. Edition d’un contact existant
C. Suppression d’un contact
D. Affichage de tous les contacts
E. Recherche d’un contact selon certains critères
F. Tri et regroupement multicritères des contacts
5. Ajout des fonctionnalités suivantes (optionnelles):
1. Login
2. Ajout d'autres tables (Adresses, Messages, Favoris, Activités…)
3. Gestion des rôles (utilisateur, gestionnaire, administrateur…)
4. Envoi de messages à des contacts
5. …

Travail pratique en JavaScript - Séquence

Travail pratique - Séquence Objectif: - Concevoir un jeu permettant de pratiquer votre mémoire. Déroulement de la partie: - A l'ouverture de l'application - Un bouton intitulé "Lancer la partie" est disponible. - Le message suivant s'affiche: Utilisez le bouton "Lancer la partie" pour débuter. - Le niveau de difficulté est par défaut au niveau normal. - Le nombre de séquence réussi pour la partie en cours est: Total 0 - Le nombre de séquence maximum ayant été réussi lors d'une des parties précédentes est: Record 0 - Après avoir cliqué sur le bouton: "Lancer la partie" - Le bouton "Lancer la partie" devient: "Stopper la partie" - Le message suivant s'affiche pendant la présentation d'une séquence aléatoire: Présentation de la séquence - Le message suivant s'affiche lorsque la présentation de la séquence se termine: À votre tour de reproduire la séquence - Une minuterie de quelques secondes démarre et une balise progress représente l'avancement de la minuterie. - Si la minuterie se termine la partie est terminé, voir: Fenêtre surgissante. - Si le joueur reproduit la séquence sans erreur - La minuterie sera réinitialisée à chaque click - La séquence sera présentée à nouveau et une étape supplémentaire sera ajoutée - Le message suivant s'affiche pendant la présentation d'une séquence aléatoire: Présentation de la séquence - Le message suivant s'affiche lorsque la présentation de la séquence se termine: À votre tour de reproduire la séquence - Si le joueur manque une étape de la séquence, la partie est terminée donc voir la partit: Fenêtre surgissante. Consignes fonctionnelles: - Fenêtre surgissante - Une fenêtre surgissante bloque l'interaction possible et affiche le titre suivant: Fin de la partie - Si le total de séquence réussi est supérieur au record, le message suivant s'affiche: Félicitations, votre nouveau record est maintenant: {indiquer la valeur actuelle} - Si le total de séquence réussi est inférieur au record, le message suivant s'affiche: Vous avez déjà fait mieux, essayé encore! - Mode de difficulté difficile - Exactement le même fonctionnement que la difficulté normale, mais en utilisant 8 boutons - La séquence aléatoire - La séquence débute avec une seule étape. - À chaque fois que le joueur réussit à reproduire la séquence, la même séquence est présentée à nouveau avec une étape supplémentaire ajoutée à la fin.
